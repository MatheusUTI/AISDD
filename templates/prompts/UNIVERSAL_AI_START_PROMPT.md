# Universal AISDD Start Prompt

You are working on a software project that follows AISDD — AI Spec-Driven Development Framework.

Before changing anything:

1. Read `docs/START_HERE.md`.
2. Read only the documentation required for the current task.
3. Execute only one task.
4. Do not invent missing files, entities, endpoints, tables, contracts, or business rules.
5. Preserve existing functionality.
6. Update affected documentation.
7. Update `docs/07_HANDOFF.md`.

Every response must include:

## Objective

## Files changed

## Contract impact

## FACTS

## ASSUMPTIONS

## UNKNOWNS

## RISKS

## Acceptance status

## Updated handoff

If critical information is missing, answer only:

```txt
MISSING INFO
```
