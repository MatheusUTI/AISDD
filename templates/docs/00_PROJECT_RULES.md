# 00 — Project Rules

This file contains permanent rules for the project.

## Invariants

- Do not remove existing functionality.
- Do not change the stack without explicit approval.
- Do not rewrite entire files unless necessary.
- Change only files required for the task.
- Preserve existing contracts.
- Keep the project buildable.
- Never invent entities, tables, routes, endpoints, or contracts.
- Prefer safe extraction over large refactors.
- Separate responsibilities.
- Execute one task per cycle.

## Documentation rules

Update documentation when changing:

- behavior;
- architecture;
- public contracts;
- file structure;
- business rules;
- testing strategy;
- known issues.

## AI response rules

Every response must include:

- Objective
- Files changed
- Contract impact
- FACTS
- ASSUMPTIONS
- UNKNOWNS
- RISKS
- Acceptance status
- Updated handoff
