# 05 — Acceptance Checks

Use this checklist before accepting any change.

## General

- [ ] Only one task was executed.
- [ ] No unrelated files were changed.
- [ ] Existing functionality was preserved.
- [ ] No contracts were invented.
- [ ] Documentation was updated if needed.
- [ ] Handoff was updated.

## Build

- [ ] Project builds successfully, if applicable.
- [ ] No obvious syntax errors were introduced.

## Tests

- [ ] Relevant tests were run, if available.
- [ ] Manual checks were performed, if needed.
- [ ] Known limitations were documented.

## AI discipline

- [ ] FACTS were separated from ASSUMPTIONS.
- [ ] UNKNOWNS were listed.
- [ ] RISKS were listed.
