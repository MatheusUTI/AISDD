# Getting Started with AISDD

## 1. Copy the documentation template

Copy `templates/docs/` into your project as `docs/`.

## 2. Fill the minimum required files

Start with:

- `docs/START_HERE.md`
- `docs/00_PROJECT_RULES.md`
- `docs/01_PRODUCT_SPEC.md`
- `docs/02_ARCHITECTURE.md`
- `docs/04_NEXT_TASK.md`
- `docs/07_HANDOFF.md`
- `docs/09_FILE_INDEX.md`

## 3. Start the first AI cycle

Use:

```txt
Read docs/START_HERE.md and execute docs/04_NEXT_TASK.md under AISDD rules.
```

## 4. Keep cycles small

Do not ask the AI to implement many unrelated features at once.

## 5. Update documentation

Every relevant change must update the affected documentation.

## 6. Continue across sessions

At the start of a new session, the AI should read:

- `docs/START_HERE.md`
- `docs/00_PROJECT_RULES.md`
- `docs/07_HANDOFF.md`
- `docs/09_FILE_INDEX.md`
- `docs/04_NEXT_TASK.md`
