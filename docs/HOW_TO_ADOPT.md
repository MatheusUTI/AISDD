# How to Adopt AISDD

This page is kept as a compatibility entrypoint.

For the current adoption model, use:

| Situation | Start here |
|---|---|
| You have a real problem but little structure | [`PROBLEM_FIRST_START.md`](PROBLEM_FIRST_START.md) |
| You want an adaptive LLM interview | [`INTERVIEW_MODE.md`](INTERVIEW_MODE.md) |
| You are a developer starting or organizing a project | [`GETTING_STARTED.md`](GETTING_STARTED.md) |
| You already have a messy or fragile project | [`RECOVERY_MODE.md`](RECOVERY_MODE.md) |
| You want to know what to copy | [`ADOPTION_PACKAGES.md`](ADOPTION_PACKAGES.md) |

## Adoption rule

Do not adopt the whole framework at once.

Choose the smallest AISDD path that reduces your next real pain.

AISDD exists to turn AI conversations into lasting software project memory, not to create documentation bureaucracy.
