# How to Adopt AISDD in an Existing Project

## Step 1 — Add AISDD docs

Copy `templates/docs/` into the project.

## Step 2 — Fill current reality

Do not describe the ideal project first.

Start by documenting the real current state in:

- `03_CURRENT_STATE.md`
- `08_KNOWN_ISSUES.md`
- `09_FILE_INDEX.md`

## Step 3 — Define permanent rules

Fill:

- `00_PROJECT_RULES.md`

Include stack, conventions, forbidden changes, and response format.

## Step 4 — Define the next task

Fill:

- `04_NEXT_TASK.md`

The file must contain only one executable task.

## Step 5 — Run the first controlled AI cycle

Ask the AI to read `START_HERE.md` and execute the next task.

## Step 6 — Maintain the system

After each cycle:

- update affected docs;
- update handoff;
- record decisions;
- keep known issues updated.
